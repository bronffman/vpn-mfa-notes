# Connectivity Checks

Сетевые проверки.

## Проверка DNS

Проверить, разрешается ли адрес VPN-сервера.

```powershell
nslookup <VPN_SERVER>
```

Ожидаемый результат:

- DNS возвращает IP-адрес;
- отсутствуют ошибки разрешения имени.

Если имя не разрешается:

- проверить подключение к интернету;
- проверить используемый DNS-сервер;
- попробовать другую сеть.

---

## Проверка доступности сервера

Проверить доступность сервера по сети.

```powershell
ping <VPN_SERVER>
```

Примечание:

Некоторые серверы не отвечают на ICMP-запросы. Отсутствие ответа на ping не всегда означает недоступность сервера.

---

## Проверка порта

Проверить доступность порта, используемого VPN.

```powershell
Test-NetConnection <VPN_SERVER> -Port 443
```

или

```powershell
Test-NetConnection <VPN_SERVER> -Port <VPN_PORT>
```

Проверить значение:

```text
TcpTestSucceeded : True
```

---

## Проверка маршрута

Если сервер недоступен, проверить маршрут.

```cmd
tracert <VPN_SERVER>
```

Обратить внимание:

- на каком узле происходит обрыв;
- есть ли потеря маршрута;
- нет ли блокировки на стороне сети.

---

## Проверка активных VPN-подключений

Убедиться, что пользователь не подключён к другому VPN.

PowerShell:

```powershell
Get-VpnConnection
```

Проверить:

- нет ли активных подключений;
- нет ли конфликтующих VPN-профилей.

---

## Что указать при эскалации

- DNS (nslookup);
- доступность порта;
- результат tracert;
- наличие доступа в интернет;
- текст ошибки;
- какие действия уже были выполнены.
