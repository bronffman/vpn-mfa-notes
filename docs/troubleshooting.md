# VPN checklist

Порядок первичной проверки, если пользователь не может подключиться к VPN.

## 1. Проверить доступ

- Пользователь должен быть добавлен в нужную группу доступа.
- Если доступ должен выдаваться через систему заявок, проверить, был ли ресурс выдан.
- Если ресурс есть, но группы нет — добавить временно вручную и передать заявку на разбор причины.

В документации не указываются реальные названия групп. Используются шаблоны:

```text
<VPN_USERS_GROUP>
<VPN_EMERGENCY_GROUP>
```

## 2. Проверить наличие VPN-профиля

Если нужного подключения нет в списке VPN, запустить восстановление профилей.

```text
Пуск → Восстановить VPN
```

После восстановления проверить, появился ли профиль:

```text
<VPN_MFA_PROFILE>
```

## 3. Проверить резервное подключение

Если основной профиль не подключается, проверить резервный профиль:

```text
<VPN_MFA_EMERGENCY_PROFILE>
```

Это помогает понять, проблема в конкретном профиле или в сети/доступе.

## 4. Проверить сеть

Типовые причины:

- нет доступа в интернет;
- пользователь уже подключён к другому VPN;
- пользователь находится во внутренней сети;
- провайдер или роутер блокирует VPN-трафик;
- DNS не может разрешить адрес VPN-сервера.

## 5. Проверить MFA

Перед подключением пользователь должен открыть приложение MFA на телефоне.
Если push не приходит — перейти к файлу `mfa-push.md`.

## 6. Эскалация

Если все пункты выше проверены, но подключение не работает, передать заявку на следующий уровень поддержки.

В комментарии нужно указать:

- какой профиль проверяли;
- есть ли доступ в интернет;
- приходит ли MFA push;
- какая ошибка отображается;
- что уже было сделано.

# MFA push не приходит

Короткий порядок проверки, если пользователь нажал подключение к VPN, но подтверждение на телефон не пришло.

## Проверить приложение

Попросить пользователя открыть приложение MFA заранее, до нажатия кнопки подключения.

Иногда push не появляется, если приложение закрыто или телефон находится в режиме энергосбережения.

## Проверить регистрацию устройства

Нужно убедиться, что у пользователя зарегистрирован аутентификатор.

Вместо реальной ссылки используется шаблон:

```text
<MFA_USER_PORTAL_URL>
```

В списке должен быть хотя бы один активный аутентификатор.

## Проверить интернет на телефоне

- Wi-Fi или мобильная сеть включены;
- приложение MFA не заблокировано;
- уведомления разрешены;
- время на телефоне выставлено корректно.

## Если всё настроено верно

Передать заявку на следующий уровень поддержки и указать:

```text
Профиль VPN: <VPN_PROFILE>
Push MFA: не приходит
Аутентификатор: зарегистрирован / не зарегистрирован
Интернет на телефоне: есть / нет
Ошибка на ПК: <текст ошибки>
```
